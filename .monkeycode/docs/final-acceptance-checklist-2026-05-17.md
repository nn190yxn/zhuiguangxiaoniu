# 追光小牛现网修复收官验收清单

## 1. 验收目标

本清单用于对本轮已完成的现网修复进行人工回归验收，重点确认以下四类结果：

1. 页面门禁是否按预期生效
2. 角色前置拦截是否与后端权限口径一致
3. 历史别名页是否都已正确跳转到正式入口
4. 正式页在未登录、普通员工、管理角色三种场景下行为是否符合预期

## 2. 验收前提

验收前准备以下三类账号或状态：

1. 未登录浏览器会话
2. 普通员工账号
3. 总部/管理角色账号

如条件允许，建议再准备：

1. 超管账号
2. 店长账号

## 3. 通用通过标准

每一页至少确认以下项目：

1. 页面能正常打开或正确跳转
2. 未登录时不会出现异常白屏、死循环跳转或控制台明显报错
3. 普通员工不会进入不该进入的后台壳或总部工具页
4. 总部/超管/店长在有权限时可正常进入并加载数据
5. 页面内关键请求仍带正确认证头，不因本轮收口造成功能失效

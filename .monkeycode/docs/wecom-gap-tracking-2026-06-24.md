## 3. 状态定义

- `pending`：已发现，尚未开始处理
- `in_progress`：处理中
- `blocked`：当前被外部条件阻塞
- `done`：已完成并已复核

## 3.1 录入规则

1. 一位员工在同一批次只保留一条记录。
2. 员工跨批次出现时，允许分别记录，但处理动作和结果要保持一致口径。
3. `当前状态` 只使用 `pending`、`in_progress`、`blocked`、`done` 四个值。
4. `处理动作` 使用结果导向描述，例如“补齐 user_id 并复核归属一致”“补绑 wecom_userid 并验证一键登录”。
5. `结果` 使用收口描述，例如“已处理并通过复核”“等待线上配置完成后复核”。
6. 每次状态变化都要同步更新 `更新时间`。
7. 如一条记录关闭，保留原记录并把状态改为 `done`，不删除历史项。

## 3.2 后台清单录入步骤

1. 打开企业微信后台一致性区，先选定本次处理范围。
2. 使用当前筛选条件导出待处理清单，确认筛选范围、审计日期和人数统计。
3. 按批次 A、B、C 的归类规则，把员工逐条录入对应批次登记表。
4. 首次录入时统一写 `当前状态 = pending`，补上员工基础信息。
5. 开始实际处理后，再把对应记录改为 `in_progress`、`blocked` 或 `done`。
6. 每处理完一批，更新该批次的汇总统计和“今日更新记录”。
7. 每天结束前复核一次：台账人数、后台导出人数、批次汇总人数三者是否一致。

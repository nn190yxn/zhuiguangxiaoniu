# 企业微信数据库变更后现网站与 H5 回归检查表

日期：2026-06-24

## 1. 目的

- 确保企业微信数据库变更执行后，现有网站、H5 和原微信小程序主链路保持正常使用。
- 只有旧链路回归通过，才进入企业微信同步、企业微信登录和企业微信消息联调。

## 2. 执行时机

1. 执行 `wecom-online-schema-2026-06-24.sql` 之后立即执行。
2. 补齐 `WECOM_*` 配置之前先做一次。
3. 挂载 worker 之前再做一次。
4. 如任一回归失败，停止继续企业微信联调，先排查现网站与 H5 影响范围。

## 3. 核心通过标准

1. 页面可打开。
2. 接口可返回。
3. 不出现新增 500 错误。
4. 不出现旧登录链路失效。
5. 不出现原提醒链路失效。

## 4. 账号与登录链路

### 4.1 H5 账号密码登录

- 接口：`POST /api/auth-jwt.php?action=login`
- 预期：账号密码登录成功，返回 `token`，现有 H5 正常进入。

### 4.2 原微信小程序登录

- 接口：`POST /api/auth-jwt.php?action=wxlogin`
- 预期：原微信登录链路保持正常，不因新增 `wecom_*` 字段失效。

### 4.3 原微信小程序绑定

- 接口：`POST /api/auth-jwt.php?action=wxbind`
- 预期：原微信绑定链路保持正常，`openid`、`openid_bound_at` 正常写入。

### 5.2 学习中心

- 接口：`/api/todos/my.php`
- 预期：必修课程待办查询正常返回。

### 5.3 制度中心

- 接口：制度列表、制度详情、制度通知相关接口

### 5.4 工作量日报

- 接口：工作量日报列表、详情、提交、证据相关接口
- 预期：日报查询和写入链路正常。

### 5.5 通知与站内提醒

- 表和接口：`mini_user_notifications`、现有通知列表接口

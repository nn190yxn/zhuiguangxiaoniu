# 常态化工作量与经营系统范围 V1

## 目标

把当前以活动结果为主的录入方式，收口为长期可用的日常经营系统。

系统目标分为四层：

- 行为层：记录员工当天实际做了什么。
- 过程层：记录邀约、沟通、上课、触达等推进过程。
- 结果层：记录成交、续费、营收等最终结果。
- 自动计算层：统一生成完成率、转化率、覆盖率、出勤率等指标。

## 范围边界

V1 先覆盖两个一线岗位：

- 销售
- 教练

V1 同时覆盖两个管理视角：

- 门店视角
- 总部运营视角

V1 暂不纳入：

- 周年庆活动型专属字段
- 抖音专题运营排期
- 非结构化复盘长文
- 薪酬计算

## 核心原则

- 一个岗位一套日录入口径，不混写。
- 结果指标尽量由过程数据自动汇总，不重复人工录入。
- 所有指标都必须绑定口径、单位、是否必填、是否允许手工覆盖。
- 所有门店、员工、岗位均复用现有 `stores`、`staffs`、`wp_users` 体系。

## 模块划分

### 1. 指标字典中心

职责：统一定义系统中的可录入指标和可计算指标。

包含字段：

- 指标编码
- 指标名称
- 岗位
- 分类
- 单位
- 数据类型
- 是否必填
- 是否系统计算
- 默认值
- 校验规则

### 2. 日报模板中心

职责：按岗位定义每天应录入哪些字段、展示顺序和校验规则。

V1 模板：

- 销售日报模板
- 教练日报模板

### 3. 日报录入中心

职责：员工按日期、门店、岗位提交当天数据。

记录维度：

- 日期
- 门店
- 员工
- 岗位
- 指标值
- 提交状态
- 提交时间

### 4. 计算汇总中心

职责：从日报明细生成门店、员工、岗位、时间维度的汇总数据。

V1 自动计算示例：

- 销售邀约完成率 = 实际邀约 / 计划邀约
- 销售到店率 = 实际到店 / 实际邀约
- 销售成交率 = 成交人数 / 实际到店
- 教练耗课完成率 = 实际耗课 / 计划耗课
- 教练沟通完成率 = 实际沟通 / 计划沟通
- 教练续费覆盖率 = 续费沟通人数 / 在读学员数

### 5. 经营看板中心

职责：按员工、门店、总部三个层级呈现结果。

V1 看板分层：

- 员工个人页
- 门店日报页
- 总部汇总页

## V1 岗位口径

### 销售

行为指标：

- 新增资源数
- 外呼数
- 微信触达数
- 跟进人数

过程指标：

- 计划邀约数
- 实际邀约数
- 计划到店数
- 实际到店数
- 计划体验数
- 实际体验数

结果指标：

- 成交人数
- 新签金额
- 续费金额

### 教练

行为指标：

- 备课次数
- 家长沟通人数
- 体测人数
- 续费沟通人数

过程指标：

- 计划耗课节数
- 实际耗课节数
- 计划沟通人数
- 实际沟通人数

结果指标：

- 暑期营推荐人数
- 续费单数
- 续费金额

## 数据约束

- 一个员工每天同一岗位同一门店只能有一份日报主记录。
- 一条日报可以包含多个指标值明细。
- 已提交日报允许二次更新，但必须保留更新时间。
- 所有金额字段单位统一为元。
- 所有率值字段不允许人工直接填写，只能系统计算。
- 如果员工跨店工作，必须允许同一天多门店录入，但每个门店独立成单。

## 首期交付建议

1. 先上线销售/教练日报录入。
2. 再上线门店汇总与总部汇总。
3. 最后补模板配置和异常预警。

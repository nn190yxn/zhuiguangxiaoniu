# 搜索专项优化方案与备案记录

## 背景

用户反馈企业内网搜索“什么都没有内容”。生产接口抽样显示常见关键词有返回，但网页、小程序制度搜索、小程序知识搜索使用多套查询和展示逻辑，登录态异常、权限过滤和局部 SQL 异常都会表现为空结果。

## 目标

- 统一网页和小程序搜索的数据口径。
- 搜索接口按模块容错，单个模块异常不影响其他模块返回。
- 空状态明确区分未登录、无匹配、接口异常和权限过滤。
- 制度、知识、课程、培训、话术、演练、考试和员工结果可直达详情或对应页面。

## 影响面

- 网页入口：`/search.html`
- 小程序入口：`mini-program/pages/policy-search/index.*`、`mini-program/pages/knowledge/list.*`
- 后端接口：`/api/search/global.php`、`/api/policy/search.php`、`/api/knowledge/list.php`
- 新增服务：`/api/search/search-service.php`

## 实施步骤

1. 新增搜索服务，集中处理关键词扩展、角色上下文、权限过滤、分类统计和模块容错。
2. 改造全局搜索接口，返回统一的 `tabs`、`results`、`total`、`empty_reason`。
3. 保持现有分组结果结构，兼容旧网页渲染。
4. 优化网页搜索空状态、登录跳转和错误提示。
5. 小程序制度搜索接入全局搜索中的制度结果。
6. 小程序知识搜索补齐搜索态提示，避免空结果被理解为系统无内容。
7. 生产部署前备份相关文件，部署后用真实账号和关键词矩阵内测。

## 回滚方案

- 恢复生产备份目录中的同名文件。
- 新增文件 `api/search/search-service.php` 保留不影响旧入口；如需完整回滚，可让 `global.php` 恢复旧版本后停止引用该文件。
- 本次优化不改数据库结构。

# 追光小牛 · 店长数据管理体系

> **版本**：v5.3 | **日期**：2026年4月
> **适用对象**：全体店长
> **核心原则**：数据不是用来汇报的，是用来发现问题和行动的。店长经营闭环总则见 `03H`。

---

## 一、制度

### 1.1 红线

| 红线 | 违反后果 |
|------|---------|
| 数据必须来自GYMGO系统和财务系统，禁止估算 | 数据造假→首次书面警告+乐捐进成长基金（员工10元/店长20元），二次辞退 |
| 每天必须查6个核心经营信号 | 不查→店长书面警告 |
| 异常数据必须当天有应对动作 | 不处理→店长书面警告 |
| 周报/月度复盘必须按时提交 | 不按时→月度考核扣分 |

### 1.2 数据管理的核心目的

| 目的 | 说明 |
|------|------|
| 发现问题 | 在问题变大之前发现它 |
| 数据是给自己看的 | 每个数字背后都对应一个动作 |
| 用数据驱动团队 | 带着"下一步怎么干"通报数据 |
| 每天看关键数字 | 抓住核心，不被数字淹没 |
| 数据异常=行动信号 | 冷静分析再行动 |

## 二、要求

### 2.1 七大数据维度

| 维度 | 定义 | 数据来源 | 核心作用 |
|------|------|---------|---------|
| **业绩数据** | 新签/续费/转介绍金额 | GYMGO系统+财务 | 决定生死 |
| **运营数据** | 到店人次/出勤率/耗课 | GYMGO系统 | 过程指标 |
| **团队数据** | 在读人数/流失预警/投诉 | GYMGO系统 | 提前干预 |
| **教学数据** | 升班率/续报率/体验转化 | GYMGO系统 | 质量把控 |
| **获客数据** | 新增资源数/资源来源/一访到店/转介绍 | GYMGO系统+手动登记 | 资源保障 |
| **线上平台数据** | 美团经营分/评价评分/抖音视频数据/抖音经营分 | 美团后台+抖音后台 | 线上门面 |
| **团队工作量数据** | 每人每日工作量完成率/周达标率/月达标率 | 门店群记录 | 执行力保障 |

### 2.2 数据检查要求

| 检查类型 | 频率 | 时间要求 | 检查内容 |
|---------|------|---------|---------|
| 日数据检查 | 每天 | 班前会前30分钟 | 6个核心经营信号 |
| 周报 | 每周一 | 12:00前 | 本周数据+复盘+计划 |
| 月度复盘 | 每月最后一天 | 22:00前 | 月度KPI+分析+下月计划 |

## 三、标准

### 3.1 每天必须看的6个核心经营信号

| # | 信号 | 在哪查 | 异常判断 | 标准动作 |
|---|------|--------|---------|---------|
| 1 | 昨日到店与今日体验 | GYMGO→今日看板/体验预约 | 到店率<80%或今日无体验 | 班前会明确邀约补量和接待分工 |
| 2 | 昨日耗课与今日课表 | GYMGO→耗课统计/排课系统 | 单日耗课<目标×70%或空堂>2节 | 当天补排课、补代课 |
| 3 | 新签进度 | GYMGO→业绩看板/资源管理 | 月进度<时间进度×80% | 确认顾问今日主攻名单和卡点客户 |
| 4 | 续费窗口覆盖 | GYMGO→到期会员/续费看板 | 未来30天到期会员有遗漏 | 确认教练跟进名单，店长标记风险会员 |
| 5 | 风险会员状态 | GYMGO→客户管理/投诉退费记录 | 应跟未跟>3个或出现投诉退费 | 店长当天升级介入 |
| 6 | 资源池存量 | GYMGO→资源管理 | 一访<周目标×30%或资源池<2周用量 | 48小时内启动补量动作 |

### 3.2 日数据检查合格标准

| 检查项 | 及格标准 |
|--------|---------|
| 检查时间 | 班前会前30分钟内完成 |
| 检查内容 | 必须完整看完6个核心经营信号 |
| 异常标记 | 任何异常必须当天有应对动作 |
| 记录方式 | GYMGO系统截图或纸质记录 |

### 3.3 异常三级预警标准

#### 红色预警（必须当天处理）

| 预警触发条件 | 标准动作 |
|-------------|---------|
| 单日体验取消/无故缺席>30% | 立即排查预约流程，当天补打邀约电话 |
| 同一时段连续3天耗课率<60% | 查教练排课/家长反馈，当天出调整方案 |
| 出现家长投诉（任何级别）| 当天必须联系家长处理 |
| 教练突发离职/请假且无替补 | 立即启用代课机制，2小时内解决 |
| 月度续费率<35% | 立即启动续费专项，组织教练逐个联系到期会员，店长跟进卡点会员 |
| 本周一访到店数为0 | 立即排查资源池，当天启动外联/转介绍/市场活动补充资源 |
| 资源池存量不足1周用量 | 立即制定资源获取计划，2天内启动外联拓展 |

#### 黄色预警（3天内处理）

| 预警触发条件 | 标准动作 |
|-------------|---------|
| 周度新签进度<时间进度×60% | 3天内密集约谈意向客户 |
| 月度续费率<目标×80% | 梳理续费窗口名单，确认教练主动触达 |
| 某教练月度耗课<目标×80% | 店长和教练面谈 |
| 连续2周到店率低于80% | 排查约课流程 |
| 本月转介绍成交为0 | 3天内排查转介绍流程，制定激励方案 |
| 本月无新增外联资源 | 3天内安排周边资源拓展（幼儿园/社区/商户） |
| 美团经营分<80分或评分<4.8分 | 3天内制定提升方案（邀约好评、优化回复、提升服务分） |
| 团队工作量周达标率<70% | 3天内排查原因，安排未达标员工面谈并制定帮扶计划 |

#### 绿色提示（本周内优化）

| 提示项 | 标准动作 |
|--------|---------|
| 体验预约充足但转化率偏低 | 本周内复盘体验接待SOP |
| 某课程满班率偏低 | 对齐原因，商量对策 |
| 团队士气一般 | 开一次非正式团队聊天 |
| 本月尚未组织会员活动 | 本周内策划一场小型会员活动（亲子课/户外体验等） |
| 本月未做顾客满意度调查 | 本周内完成至少5份满意度调查（面谈或线上问卷） |
| 本周抖音视频发布不足5条 | 安排补发，确保周发布量达标 |
| 抖音经营分连续2周下降 | 本周内分析原因，优化视频内容策略 |
| 本月月度好评数不足15条（美团+抖音） | 本周内安排好评邀约，确保月度达标 |

## 四、流程SOP

### 4.1 日数据检查SOP（10分钟）

```
Step 1 → 打开GYMGO系统（2分钟）
  └→ 进入今日看板，快速浏览6个核心经营信号

Step 2 → 标记异常数字（2分钟）
  └→ 哪个指标超出正常范围，用荧光笔/红笔标出

Step 3 → 初步分析原因（3分钟）
  └→ 新问题还是老问题？个人原因还是团队原因？

Step 4 → 准备班前会数据通报（3分钟）
  └→ 数据好的点→表扬团队
  └→ 数据差的点→带出"今天怎么干"

Step 5 → 异常当场处理（贯穿全天）
  └→ 有问题立即分配任务
```

### 4.2 周报填写SOP（周一30分钟）

```
Step 1 → 导出本周数据（5分钟）
Step 2 → 逐项填入周报模板（10分钟）
Step 3 → 复盘本周（10分钟）
Step 4 → 规划下周（5分钟）
Step 5 → 发门店运营群+企业微信（2分钟）
```

### 4.3 月度复盘SOP

```
Step 1 → GYMGO导出上月数据
Step 2 → 填总册第10表
Step 3 → 分析：好与不好各TOP3
Step 4 → 下月3个重点工作
Step 5 → 发企业微信给总经理/督导
```

|------|---------|--------|---------|
| 日数据检查 | 班前会前30分钟 | 店长 | GYMGO截图 |

| 月中进度检查 | 每月15号前后 | 店长 | 进度报告 |

### 5.2 考核机制

| 考核项 | 考核对象 | 考核频率 | 考核人 | 考核标准 |
|--------|---------|---------|--------|---------|
| 日数据检查 | 店长 | 每日 | 督导 | 6个核心经营信号是否检查 |
| 周报准时率 | 店长 | 每周 | 总经理 | 周一12:00前 |
| 周报数据准确性 | 店长 | 每周 | 督导 | 与GYMGO一致 |
| 月度复盘质量 | 店长 | 每月 | 总经理 | 有分析有计划 |
| 异常处理及时性 | 店长 | 每月 | 督导 | 当天处理 |

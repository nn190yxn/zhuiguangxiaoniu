> **日期**：2026年6月27日
> **参与人**：创始人 + AI Coding Agent

## 一、背景与核心问题

### 1.1 现状

- 5家门店处于"连而不锁"状态
- 教练偏懒惰、缺乏主动性、缺乏思考
- 销售团队缺乏固化行为
- 总部后台仅2人（1督导+1财务），管控力度不足
- 团队综合素质低、学历不高、执行力弱

### 1.2 核心思路

**僵化 → 固化 → 优化**。先把铁箍用铁丝箍起来，再谈谁跑得快。

下半年以**控制**为核心，不以激发为主。系统替你思考，人只负责打勾。

## 二、三条业务主线的确立

创始人提出三条核心业务线，它们交叉但不割裂：

| 业务线 | 定义 | 核心目标 |
|--------|------|---------|
| **体验线** | 新签→续费→转介绍，客户全生命周期 | 控制关键节点"做没做" |
| **交付线** | 教案→课堂→ACE评估，教学品质链 | 控制安全+对齐 |
| **人员线** | 目标分解→行为→考核→晋升 | 控制行为+工作量 |

### 交叉点示例

"续费"同时触及三条线：
- 体验线：剩余20节必须启动触达（控制做没做）
- 交付线：触达前必须有阶段评估报告（控制有没有内容可说）
- 人员线：续费达成纳入星级评定（控制与收入挂钩）

## 三、组织架构调整

### 3.1 原始架构问题

```
    总经理
    /    \
 督导(虚) 财务
   |
 5位店长(过载)
 / \
教练 顾问
```

- **督导被架空**：平级管店长，无实权来源，店长不配合
- **店长过载**：又要管人又要看课又要跑市场，全要=全不行
- **创始人缺位**：定完制度没人盯执行

### 3.2 调整后架构

```
              创始人
           （定标准/看红线/定方向）
                │
      ┌─────────┼─────────┐
      │         │          │
   总经理      财务       督导
（运营+品质控制）（数据中心）（总经理品质执行臂）
      │
 ┌──┬─┼─┬──┐
店长1 店长2 店长3 店长4 店长5
[体验线+人员线日控]
  │
[四五星教练=交付线第一道品控]
```

### 3.3 创始人-总经理分工

| 维度 | 创始人 | 总经理 |
|------|--------|--------|
| 定位 | 定标准、定方向、看红线 | 执行标准、日常运营、品质控制 |
| 看什么 | 只看连续红色预警 | 每天看红灯清单 |
| 干预时机 | 月度品质报告连续红色时 | 每天、每周、每月 |
| 不开什么会 | 不参加周例会、不审日常 | 主持周例会、月度复盘 |

### 3.4 督导重新定位

**旧**：平级检查店长，无实权 → **新**：总经理的品质执行臂，背靠总经理权威执行

关键变化：督导不再自己做决策和判断，只做事实记录和上报。所有检查行为代表总经理，店长不配合督导=不配合总经理。

### 3.5 不增编

督导保留、财务保留、五家店长保留。四五星教练兼教学品控。总编制不变。

## 四、核心控制工具——傻瓜式日控清单

针对团队素质低、执行力弱的现实，所有一线控制节点设计为Y/N判断，不要求主观判断。

### 4.1 店长日控清单（6勾，3分钟）

| 勾 | 检查内容 | 时限要求 |
|----|---------|---------|
| 1 | 昨天线上线索（抖音/美团）联系+加微信+确认时间了没 | 30min内 |
| 2 | 昨天线下线索联系了没 | 2h内 |
| 3 | 昨天体验课签到做了没 | — |
| 4 | 前天体验课后回访了没 | 24h内 |
| 5 | 有学员剩≤20节未启动续费的吗 | 每日 |
| 6 | 昨天本店抖音发布了吗 | 每日 |

### 4.2 店长周查清单（2勾，5分钟）

| 勾 | 检查内容 |
|----|---------|
| 7 | 成交满7天的，转介绍邀请发了吗 |
| 8 | 首月服务跟进表中本周该到期的节点做了吗 |

### 4.3 教案安全栏日控（1分钟）

只看教案中"安全保护措施"栏有没有字——有=过，空=追。**教练不单独写安全教案，在ACE教案中填写即可。**

### 4.4 人员工作量日控（2分钟）

每个教练/顾问昨天工作量够不够4个——翻表，够=过，不够=记名。

## 五、线上线索专项检查机制（三层交叉验证）

针对抖音/美团冲动消费型客户，30min内不联系大概率退款。只靠店长自打勾不可靠。

| 层 | 谁验 | 怎么验 | 造假后果 |
|----|------|--------|---------|
| 第一层 | 财务系统自动标红 | GYMGO拉订单时间戳，>30min自动进红灯清单 | 店长来不及造假 |
| 第二层 | 督导回拨验证 | 每店每月抽3个线上客户，回拨家长："您下单后多久有老师联系您？" | 1次假勾=书面警告+乐捐20元，2次=当月降级 |
| 第三层 | 总经理退款回溯 | 月度数据月报看线上退款率，异常追查 | 退款超均值→深度审计 |

**为什么能跑通**：系统先标红→店长没有造假窗口；督导每月抽3次回拨→店长赌不起。

## 十、服务器与GitHub代码一致性问题

### 10.1 对比结果

对349个静态文件（JS/HTML/CSS）进行了全量对比：

| 类别 | 数量 | 占比 |
|------|------|------|
| 完全一致 | 127个 | 36% |
| 仅换行符差异（±1字节） | ~170个 | 49% |
| 实质代码差异 | 4个 | 1.1% |
| 未部署或无响应 | ~48个 | 14% |

### 10.2 4个实质代码差异

| 文件 | 仓库 | 服务器 | 差异说明 |
|------|------|--------|---------|
| `mini-program/utils/api.js` | 4977B | 2484B | 超时时间从60s改为15s |
| `internal.html` | 25559B | 23752B | 内网首页内容变动 |
| `fitness-assessment-app.html` | 200192B | 195016B | 体测评估页面变动 |
| `admin/security-login-audit.html` | 12108B | 11860B | 安全审计页面变动 |

### 10.3 PHP文件无法对比

服务器上的PHP文件（153个API接口）会执行而非返回源码，无法通过HTTP对比内容。需SSH才能做完整diff。

### 10.4 结论

**98%的静态文件一致或仅格式差异。4个文件有实质代码改动，其中`mini-program/utils/api.js`的改动（超时60s→15s）确认是直接在服务器上改的，未同步回GitHub。**

## 十一、关键设计决策

| 决策 | 理由 |
|------|------|
| 所有控制节点设计为Y/N | 团队素质低，不能依赖主观判断 |
| 教练不单独写安全教案 | 降低负担，在ACE教案中填一栏即可 |
| 督导保留但不撤销 | 改为总经理品质执行臂，赋予授权背书 |
| 四五星教练做教学品控 | 不增编，存量激活 |
| 线上线索30min响应 | 抖音冲动消费，慢则退 |
| 督导每月回拨家长验证 | 威慑假勾，比制度更有效 |
| 先系统标红后人打勾 | 店长没有造假窗口 |
| 僵化期只抓4件事 | 不一口气压考核，渐进落地 |
| 核心部分控制优化，基础部分标准规范 | 区分轻重，店长不被68项检查压垮 |
| 创始人只定标准看红线，不介入日常 | 创始人不是最贵的店长 |

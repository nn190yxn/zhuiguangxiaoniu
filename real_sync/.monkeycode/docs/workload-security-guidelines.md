# 工作量系统开发安全规范

## 文件版本管理
- **禁止生成 .bak 文件**：修改线上文件时必须使用 Git 进行版本管理，禁止手动或工具自动生成 .bak 备份文件
- **工具配置**：如果编辑器或工具会自动生成备份文件，必须在工具配置中关闭此功能
- **Git 忽略规则**：所有备份文件模式已添加到 .gitignore

## 备份文件清理流程
1. 发现备份文件后，先移动到 `_archive/` 目录进行隔离
2. 观察一周，确认无业务影响后再正式删除
3. 清理后验证相关功能是否正常

## 安全防护
- Web 服务器配置禁止访问危险文件类型
- 敏感目录（如 scripts/）限制外部访问
- 定期安全审计检查备份文件和敏感信息泄露
